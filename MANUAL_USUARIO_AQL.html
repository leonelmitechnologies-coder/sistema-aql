1. Introducción

1.1 ¿Qué es el Sistema AQL?

El Sistema AQL (Acceptable Quality Level) es una aplicación web diseñada para el departamento de Control de Calidad. Permite registrar, gestionar y analizar las inspecciones de calidad realizadas a televisores durante el proceso de recepción y verificación.

El sistema opera de forma centralizada utilizando una base de datos PocketBase, lo que permite que múltiples inspectores trabajen simultáneamente desde diferentes dispositivos, compartiendo los datos en tiempo real.

1.2 ¿Para qué sirve?

  • Registrar inspecciones de televisores con datos completos (SKU, marca, modelo, clasificación, estado de componentes)
  • Evaluar calidad siguiendo el proceso AQL: caja, accesorios, protección, audio, video, pantalla y chasis
  • Determinar aceptación o rechazo de cada unidad inspeccionada
  • Consultar historial de todas las inspecciones realizadas por cualquier inspector
  • Visualizar estadísticas mediante gráficos interactivos en el Dashboard
  • Exportar datos a Excel para reportes y análisis externos
  • Trabajo multi-equipo con datos sincronizados en tiempo real

3. Registro de Inspección

El registro de cada inspección se realiza en dos pasos. Esta es la pestaña principal del sistema y la que usará con mayor frecuencia.

3.1 Paso 1: Datos de la televisión

En este paso se capturan los datos de identificación del televisor:

License Plate (Placa)

Escanee o escriba el código de placa del televisor. Este campo es obligatorio y sirve como identificador único de la unidad.

SKU

Utilice el campo de búsqueda inteligente para encontrar el SKU. Puede buscar por:

  • Código SKU (ej: SNTV007001)
  • Nombre de marca (ej: Samsung)
  • Número de modelo

Al seleccionar un SKU, los campos Marca, Modelo, Descripción y Pulgada se llenan automáticamente.

Consejo Escriba al menos 2 caracteres para que aparezcan las sugerencias. El catálogo contiene todos los SKUs registrados en el sistema.

Clasificación

Seleccione la clasificación del televisor. Las opciones disponibles son:

CódigoSignificado
GRAGrado A
GRBGrado B
GRCGrado C
ICBIncompleto B
ICCIncompleto C
ICDIncompleto D
ICXIncompleto X
BOXSolo caja

Una vez completados todos los campos obligatorios, haga clic en "Siguiente" para continuar al Paso 2.

3.2 Paso 2: Proceso AQL

En este paso se evalúa la calidad del televisor en diferentes aspectos. Todos los campos son obligatorios.

Evaluación de la caja y accesorios

CampoOpcionesDescripción
¿Clasificación coincide con caja? SI / NO Verificar que la clasificación asignada coincide con el estado real de la caja
¿Accesorios conformes? SI / NO Verificar que todos los accesorios estén presentes y en buen estado
¿Protección en buen estado? SI / NO Verificar el estado de las protecciones internas (espuma, plástico, etc.)

Evaluación de componentes

Para cada componente, seleccione uno de los tres estados posibles:

EstadoSignificadoCuándo usarlo
Excelente Sin defectos El componente funciona perfectamente, sin ningún problema visible o audible
Buen Estado Defectos menores Funciona correctamente pero con imperfecciones menores aceptables
Con Problemas Defectos significativos Presenta fallas, daños o defectos que afectan la funcionalidad

Los componentes evaluados son:

  • Audio - Funcionamiento de bocinas y salida de sonido
  • Video - Calidad de imagen, resolución, colores
  • Pantalla - Estado físico del panel (rayaduras, manchas, píxeles muertos)
  • Chasis - Estado físico de la carcasa y estructura

Resultado AQL

Finalmente, determine el resultado de la inspección:

  • ACEPTADO - La unidad cumple con los estándares de calidad
  • RECHAZADO - La unidad NO cumple con los estándares de calidad

Al hacer clic en "Guardar Registro", la inspección se almacena en el servidor y queda visible para todos los inspectores.

4. Historial de Registros

La pestaña Historial muestra todas las inspecciones realizadas por todos los inspectores del equipo.

4.1 Consultar registros

Al hacer clic en la pestaña "Historial", el sistema carga automáticamente todos los registros desde el servidor. La tabla muestra:

  • Número de registro
  • Fecha y hora de la inspección
  • License Plate, SKU, Marca, Modelo, Pulgada
  • Clasificación
  • Estado de cada componente evaluado (con indicadores de color)
  • Resultado AQL final

En la parte superior se muestran tres contadores:

  • Total - Número total de inspecciones
  • Aceptados - Inspecciones con resultado ACEPTADO
  • Rechazados - Inspecciones con resultado RECHAZADO

4.2 Filtrar registros

Puede filtrar los registros utilizando los controles de búsqueda en la parte superior de la tabla:

FiltroFunción
Campo de texto Buscar por License Plate, SKU, marca o modelo
Filtro AQL Mostrar solo ACEPTADO, solo RECHAZADO, o todos
Filtro Clasificación Mostrar solo una clasificación específica (GRA, GRB, etc.)
Consejo Los filtros se aplican en tiempo real mientras escribe o selecciona opciones. Cuando está conectado al servidor, los filtros se ejecutan directamente en PocketBase para mayor velocidad.

4.3 Exportar a Excel

Para exportar todos los registros a un archivo Excel:

  1. Haga clic en el botón "Exportar Excel"
  2. Se descargará automáticamente un archivo AQL_Registros_YYYY-MM-DD.xlsx
  3. El archivo contiene todos los campos de cada inspección, incluyendo el nombre del inspector

4.4 Eliminar registros

  • Eliminar uno: Haga clic en "Eliminar" en la fila correspondiente. Se pedirá confirmación.
  • Eliminar todos: Haga clic en "Limpiar Todo". Esta acción elimina todos los registros del servidor y no se puede deshacer.
Precaución Eliminar registros es una acción permanente. Asegúrese de exportar los datos a Excel antes de eliminar si necesita conservar la información.

5. Dashboard

La pestaña Dashboard ofrece una vista gráfica de los datos de inspección para facilitar el análisis y la toma de decisiones.

5.1 Indicadores KPI

En la parte superior del Dashboard se muestran cuatro indicadores clave:

KPIDescripción
Total Inspecciones Cantidad total de inspecciones registradas en el sistema
Aceptados verde Cantidad de inspecciones con resultado ACEPTADO
Rechazados rojo Cantidad de inspecciones con resultado RECHAZADO
Tasa de Aceptación azul Porcentaje de unidades aceptadas sobre el total (Aceptados / Total x 100)

5.2 Gráficos

El Dashboard incluye seis gráficos interactivos:

Estado AQL (Gráfico de dona)

Muestra la proporción de televisores aceptados vs rechazados. Permite ver de un vistazo la tasa general de calidad.

Por Clasificación (Gráfico de barras)

Muestra cuántas inspecciones se han realizado por cada tipo de clasificación (GRA, GRB, GRC, etc.). Útil para identificar qué clasificaciones llegan con mayor frecuencia.

Estado de Componentes (Barras apiladas horizontales)

Muestra el estado de los cuatro componentes evaluados (Audio, Video, Pantalla, Chasis) apilando los tres estados posibles (Excelente, Buen Estado, Con Problemas). Permite identificar rápidamente qué componente presenta más problemas.

Inspecciones por Día (Gráfico de línea)

Muestra la tendencia diaria de inspecciones. Permite ver el volumen de trabajo a lo largo del tiempo y detectar picos o caídas en la actividad.

Por Marca (Gráfico de barras)

Muestra cuántas inspecciones se han realizado por cada marca de televisor. Útil para ver la distribución del inventario inspeccionado.

Por Inspector (Gráfico de dona)

Muestra la distribución del trabajo entre los inspectores. Permite ver cuántas inspecciones ha realizado cada miembro del equipo.

Consejo Los gráficos se actualizan cada vez que abre la pestaña Dashboard. Los datos se cargan directamente desde el servidor para mostrar siempre la información más reciente.

6. Funcionamiento Multi-equipo

El sistema está diseñado para que múltiples inspectores trabajen simultáneamente, compartiendo datos en tiempo real.

6.1 Sincronización en tiempo real

Inspector A
registra TV
→
Servidor
PocketBase
→
Inspector B
ve el registro

Cuando un inspector guarda una inspección:

  1. El registro se envía inmediatamente al servidor PocketBase
  2. El servidor notifica a todos los inspectores conectados (realtime)
  3. Si otro inspector está viendo el Historial, los datos se actualizan automáticamente

Esto significa que todos los inspectores siempre ven los mismos datos, sin importar quién los haya registrado.

6.2 Modo sin conexión

Si se pierde la conexión con el servidor durante una inspección:

  1. Los datos se guardan localmente en el navegador
  2. Se agregan a una cola de pendientes de sincronización
  3. Cuando se recupera la conexión, los datos pendientes se envían automáticamente al servidor
Nota Los datos guardados localmente solo existen en ESE navegador/dispositivo. No serán visibles para otros inspectores hasta que se sincronicen con el servidor.

6.3 Barra de estado

En la parte superior de la aplicación, la barra de estado indica:

IndicadorColorSignificado
Conectado Verde Conexión activa con el servidor. Los datos se sincronizan normalmente.
Sincronizando Amarillo Enviando o recibiendo datos del servidor.
Desconectado Rojo Sin conexión al servidor. Los datos se guardan localmente.
